üí° Key Concepts

1. Air-Gapped Networks

What is an Air-Gapped Network?

An air-gapped network is physically isolated from unsecured networks, including the internet. It creates a "gap" of air between the secure network and potential threats.

Why Use Air-Gapping?
  • Maximum Security: Eliminates entire classes of network-based attacks
  • Compliance: Required for certain industries (defense, healthcare, finance)
  • Intellectual Property: Protects sensitive data and trade secrets
  • Critical Infrastructure: Essential for SCADA and industrial control systems
Real-World Examples:
  • Military classified networks
  • Nuclear facility control systems
  • Financial trading systems
  • Healthcare patient data systems

2. VPN and WireGuard Protocol

Virtual Private Networks (VPN)

A VPN creates a secure, encrypted tunnel between a client and a private network over the internet.

WireGuard Advantages:
  • Modern Cryptography: Uses state-of-the-art protocols (ChaCha20, Poly1305)
  • Simplicity: Only 4,000 lines of code vs 100,000+ in IPSec
  • Performance: Significantly faster than OpenVPN and IPSec
  • Security: Smaller attack surface, easier to audit
# WireGuard Connection Flow Client ‚Üí Public Internet ‚Üí VPN Gateway ‚Üí Private Network (Encrypted) (Authenticated)

3. PKI and Certificate Chains

Public Key Infrastructure (PKI)

PKI is a framework of roles, policies, and procedures needed to create, manage, and revoke digital certificates.

3-Tier Certificate Hierarchy:
  1. Root CA: The ultimate trust anchor, kept offline for maximum security
  2. Intermediate CA: Signs server certificates, can be revoked without affecting root
  3. Server Certificate: Identifies the actual service (e.g., jenkins.example.com)
Why Three Tiers?
  • Security: Root CA compromise doesn't require re-trusting all clients
  • Flexibility: Can create multiple intermediate CAs for different purposes
  • Revocation: Can revoke intermediate without affecting other certificates
  • Industry Standard: Matches practices of commercial CAs

4. Defense in Depth

Layered Security Architecture

Defense in depth uses multiple security layers so that if one fails, others still protect the system.

5 Security Layers in This Project:
  1. Network Isolation: No public IPs, no direct internet access
  2. VPN Authentication: Only authenticated users can reach the network
  3. Firewall Rules: GCP firewalls with deny-all default policy
  4. TLS Encryption: All traffic encrypted with strong ciphers
  5. Application Security: Jenkins authentication and role-based access

Layer 2: VPN
Layer 3: Firewall
Layer 4: TLS
Layer 5: Application

5. Infrastructure as Code (IaC)

Terraform for Cloud Infrastructure

IaC treats infrastructure configuration as software code, enabling version control, testing, and automation.

Benefits:
  • Reproducibility: Deploy identical environments every time
  • Version Control: Track changes and roll back if needed
  • Documentation: Code serves as living documentation
  • Collaboration: Teams can review and approve changes

Lab 2: Certificate Management

üîê Practice: Create Your Own PKI

Duration: 60 minutes

Tools: OpenSSL

Step 1: Generate Root CA private key and certificate
openssl genrsa -out root-ca.key 4096 openssl req -x509 -new -nodes -key root-ca.key -sha256 -days 3650 -out root-ca.crt
Step 2: Create Intermediate CA
openssl genrsa -out intermediate-ca.key 4096 openssl req -new -key intermediate-ca.key -out intermediate-ca.csr openssl x509 -req -in intermediate-ca.csr -CA root-ca.crt -CAkey root-ca.key -CAcreateserial -out intermediate-ca.crt -days 1825
Step 3: Generate server certificate
Step 4: Verify the certificate chain
openssl verify -CAfile root-ca.crt -untrusted intermediate-ca.crt server.crt
Step 5: Install root CA in your browser/system
Step 6: Configure a web server with your certificate
