How Encryption Actually Works (Public Key Cryptography)

This is where it gets interesting! Certificates use asymmetric encryption:

Public & Private Keys
üîì
Public Key

‚úÖ Everyone can see it
‚úÖ In the certificate
‚úÖ Used to ENCRYPT data

+
üîê
Private Key

üîí Secret! Only server has it
üîí Never shared
üîí Used to DECRYPT data

The Magic:

  • Data encrypted with PUBLIC key can ONLY be decrypted with PRIVATE key
  • This means you can send secrets to a website even over an untrusted network
  • Hackers can see the encrypted data but can't decrypt it without the private key

Common Certificate Issues & Troubleshooting

üî¥
Certificate Expired

Why: Certificates have expiration dates (usually 90 days to 1 year)

Fix: Renew the certificate before expiration

üî¥
Hostname Mismatch

Why: Certificate issued for "www.example.com" but you're accessing "example.com"

Fix: Use a certificate with multiple SANs (Subject Alternative Names)

üî¥
Untrusted Certificate Authority

Why: Certificate signed by unknown or self-signed CA

Fix: Get certificate from a trusted public CA

üî¥
Incomplete Certificate Chain

Why: Intermediate certificates not installed on server

Fix: Install the full certificate chain (root + intermediate + end entity)

üí° Challenge: Why use asymmetric encryption for handshake then switch to symmetric?

Answer: Great question! Here's why:

  • Asymmetric (Public/Private Key): Very secure but SLOW
  • Symmetric (AES): Very fast but requires both parties to have the same key
  • Solution: Use asymmetric encryption to securely exchange a symmetric key, then use that fast symmetric key for actual data transfer
  • Analogy: It's like using a heavy safe to transport a lightweight key, then using that key for everyday operations

Certificate Pinning

What: Hard-coding or pre-loading the expected certificate or public key in your application

Why: Prevents man-in-the-middle attacks even if a CA is compromised

Used by: Banking apps, high-security applications

How It Works:

  • App stores the hash of the expected certificate
  • When connecting, app checks if the server's cert matches the stored hash
  • If it doesn't match, connection is rejected even if cert is valid
‚ö†Ô∏è

Risk: If you pin and then need to change certificates (expiration, compromise), you must update all apps. Users on old versions will be locked out!
