WHAT IS DNS?

Beginner

DNS = The Internet's Phone Book

Imagine you want to call your friend, but you don't remember their phone number. What do you do? You look it up in your phone's contacts!

DNS (Domain Name System) works exactly like that for the internet:

üí°

The Problem: Computers communicate using IP addresses (like 192.0.2.1), but humans can't remember these numbers!

The Solution: DNS translates easy-to-remember domain names like learningmyway.space into IP addresses that computers understand.

Real-World Example:

üë§ You type: learningmyway.space
‚Üì
üîç DNS Magic Happens
(We'll learn the details next!)
‚Üì
üéØ DNS Returns: 192.0.2.123
‚Üì
üåê Your browser connects to 192.0.2.123
Website loads!

üîë KEY CONCEPTS:

  • Domain Name: Human-friendly address (learningmyway.space)
  • IP Address: Computer-friendly numbers (192.0.2.123)
  • DNS Server: The "phone book" that does the translation
  • Resolution: The process of converting domain ‚Üí IP

Understanding Domain Names

Let's break down: www.learningmyway.space

DOMAIN HIERARCHY
. (root)
The very top - usually invisible
Managed by: IANA (Internet Assigned Numbers Authority)
‚Üì
.space (TLD - Top Level Domain)
Like .com, .org, .net, .space
Managed by: Registry operators (e.g., Radix for .space)
‚Üì
learningmyway (Second Level Domain)
This is what you register and own!
Managed by: YOU (via registrar)
‚Üì

Optional prefix you can create

Managed by: YOU (in your DNS settings)

Important: You don't "own" a domain forever. You lease it for a period (usually 1-10 years) and must renew it. If you don't renew, someone else can register it!

PART 2: What Happens When Someone Visits Your Site

Complete real-time DNS resolution workflow:

COMPLETE DNS RESOLUTION (Real-Time)
üë§ USER TYPES: learningmyway.space
Browser: "I need the IP address for this!"
‚Üì
üíæ CHECK LOCAL CACHE
Browser checks: "Have I visited this recently?"
If YES ‚Üí Use cached IP (instant!)
If NO ‚Üí Continue to next step
‚Üì
üñ•Ô∏è CHECK OS CACHE
Operating System (Windows/Mac/Linux) checks its cache
Location: /etc/hosts or system resolver
If found ‚Üí Return IP
If not ‚Üí Continue
‚Üì
üîç QUERY RECURSIVE RESOLVER
Usually your ISP's DNS server
Examples: 8.8.8.8 (Google), 1.1.1.1 (Cloudflare)
"Hey, do you know learningmyway.space?"
‚Üì
üåç QUERY ROOT NAMESERVER
Recursive resolver: "Where can I find .space domains?"
Root server: "Ask the .space TLD nameservers!"
Returns: IP addresses of .space nameservers
There are 13 root server clusters worldwide
‚Üì
üè¢ QUERY TLD NAMESERVER (.space)
Recursive resolver: "Where is learningmyway.space?"
TLD server: "Ask these authoritative nameservers!"
Returns: Your domain's nameserver addresses
(e.g., ns1.cloudflare.com, ns2.cloudflare.com)
‚Üì
üì° QUERY AUTHORITATIVE NAMESERVER
Recursive resolver: "What's the IP for learningmyway.space?"
Your nameserver: "Here it is: 192.0.2.123"
This is YOUR DNS server with YOUR records!
‚Üì
üíæ CACHE THE RESULT
Recursive resolver saves this for future queries
TTL (Time To Live): Usually 300-3600 seconds
Next query will be instant (from cache)!
‚Üì
üì® RETURN TO BROWSER
Recursive resolver ‚Üí OS ‚Üí Browser
"Here's the IP: 192.0.2.123"
‚Üì
üåê BROWSER CONNECTS
HTTP/HTTPS request to 192.0.2.123
Website loads!
Total time: 20-300ms (first visit)
Cached: <1ms (subsequent visits)
‚ö°

Speed Optimization: This entire process happens in milliseconds! Caching at multiple levels ensures most DNS lookups are nearly instant. That's why your second visit to a website is always faster!

PART 3: Nameservers Explained

Nameservers are the DNS servers that hold your domain's DNS records.

Two Main Options:

üè¢
REGISTRAR'S NAMESERVERS

Example:
ns1.namecheap.com
ns2.namecheap.com

  • Default option
  • Manage DNS in registrar dashboard
  • Good for beginners
  • Basic features
‚òÅÔ∏è
THIRD-PARTY NAMESERVERS

Example:
ns1.cloudflare.com
ns2.cloudflare.com

  • More features (CDN, DDoS protection)
  • Faster global performance
  • Better analytics
  • Requires changing nameservers

How to Change Nameservers (learningmyway.space):

Example: Switching to Cloudflare 
1. Sign up at Cloudflare.com
2. Add your domain: learningmyway.space
3. Cloudflare scans your existing DNS records
4. Cloudflare gives you new nameservers:
   - ns1.cloudflare.com
   - ns2.cloudflare.com

5. Go to your registrar (e.g., Namecheap)
6. Find "Nameservers" or "DNS" settings
7. Change from "Default" to "Custom"
8. Enter Cloudflare's nameservers
9. Save changes

10. Wait for propagation (0-48 hours, usually <4 hours)
11. Cloudflare will notify you when active
12. Now manage DNS in Cloudflare dashboard!
‚ö†Ô∏è

DNS Propagation: When you change nameservers, it takes time for the change to spread worldwide. This is because:

  • DNS servers cache old information
  • Changes must propagate through global DNS hierarchy
  • ISPs update at different rates
  • Your TTL settings affect speed

Tip: Lower your TTL to 300 seconds a day before making changes!

ALL DNS RECORD TYPES

Advanced

Complete DNS Records Guide

Let's explore EVERY DNS record type with real examples for learningmyway.space

A RECORD

Purpose: Maps domain name to IPv4 address (32-bit)

Most Common Use: Pointing your domain to a web server

Example for learningmyway.space:

Type: A
Name: @
Value: 192.0.2.123
TTL: 3600

Result: learningmyway.space ‚Üí 192.0.2.123
Subdomain Example:

Type: A
Name: www
Value: 192.0.2.123
TTL: 3600

Result: www.learningmyway.space ‚Üí 192.0.2.123
  • @ means "root domain" (learningmyway.space)
  • Can have multiple A records for same name (load balancing)
  • Most websites need at least 2 A records (@ and www)
AAAA RECORD

Purpose: Maps domain name to IPv6 address (128-bit)

Use Case: Support for modern IPv6 protocol

Example:

Type: AAAA
Name: @
Value: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
TTL: 3600

Result: learningmyway.space ‚Üí 2001:0db8:85a3::8a2e:0370:7334
üí°

Why IPv6? We're running out of IPv4 addresses (only 4.3 billion). IPv6 has 340 undecillion addresses! Always add AAAA records alongside A records for future-proofing.

CNAME RECORD

Purpose: Creates an alias from one domain to another (Canonical Name)

Use Case: Point multiple names to the same destination

Example - Pointing www to root:

Type: CNAME
Name: www
Value: learningmyway.space
TTL: 3600

Result: www.learningmyway.space ‚Üí points to learningmyway.space
Example - Subdomain to external service:

Type: CNAME
Name: blog
Value: myblog.ghost.io
TTL: 3600

Result: blog.learningmyway.space ‚Üí points to Ghost hosting
‚ö†Ô∏è

CRITICAL RULE: You CANNOT create a CNAME for the root domain (@). This is a DNS protocol limitation. Use A/AAAA records or ALIAS/ANAME records instead (if your provider supports them).

MX RECORD

Purpose: Specifies mail servers for your domain (Mail Exchange)

Use Case: Receiving email at your domain (e.g., rksuraj@learningmyway.space)

Example - Google Workspace:

Type: MX
Name: @
Priority: 1
Value: aspmx.l.google.com

Type: MX
Name: @
Priority: 5
Value: alt1.aspmx.l.google.com

Type: MX
Name: @
Priority: 10
Value: alt2.aspmx.l.google.com
  • Priority: Lower number = higher priority (try this server first)
  • Multiple MX records provide redundancy
  • If priority 1 fails, mail server tries priority 5, then 10
  • Required if you want to receive email at your domain
TXT RECORD

Purpose: Store arbitrary text data - most versatile record type!

Common Uses: Verification, email security, site ownership proof

Example 1 - Domain Verification (Google):

Type: TXT
Name: @
Value: "google-site-verification=abc123xyz789"
TTL: 3600
Example 2 - SPF (Email Authentication):

Type: TXT
Name: @
Value: "v=spf1 include:_spf.google.com ~all"
TTL: 3600

Meaning: Only Google's servers can send email from @learningmyway.space
Example 3 - DKIM (Email Signature):

Type: TXT
Name: default._domainkey
Value: "v=DKIM1; k=rsa; p=MIGfMA0GCSqGSIb3DQEBA..."
TTL: 3600
Example 4 - DMARC (Email Policy):

Type: TXT
Name: _dmarc
Value: "v=DMARC1; p=quarantine; rua=mailto:dmarc@learningmyway.space"
TTL: 3600
NS RECORD

Purpose: Specifies authoritative nameservers for your domain

Use Case: Delegates DNS control to specific nameservers

Example - Cloudflare Nameservers:

Type: NS
Name: @
Value: ns1.cloudflare.com

Type: NS
Name: @
Value: ns2.cloudflare.com
Example - Subdomain Delegation:

Type: NS
Name: api
Value: ns1.aws.com

Result: api.learningmyway.space DNS is managed by AWS
‚ö†Ô∏è

Important: NS records at the root (@) are usually set by your registrar and shouldn't be changed directly. You change nameservers in your registrar's dashboard, which updates these records automatically.

SRV RECORD

Purpose: Specifies location of specific services (Service Record)

Use Case: VoIP, gaming servers, instant messaging, XMPP

Example - Minecraft Server:

Type: SRV
Name: _minecraft._tcp
Priority: 0
Weight: 5
Port: 25565
Target: mc.learningmyway.space

Result: Players can connect via learningmyway.space instead of IP
Example - XMPP Chat:

Type: SRV
Name: _xmpp-client._tcp
Priority: 10
Weight: 0
Port: 5222
Target: chat.learningmyway.space
CAA RECORD

Purpose: Specifies which Certificate Authorities can issue SSL certificates

Use Case: Security - prevent unauthorized certificate issuance

Example - Allow Only Let's Encrypt:

Type: CAA
Name: @
Flags: 0
Tag: issue
Value: "letsencrypt.org"

Result: Only Let's Encrypt can issue certs for learningmyway.space
Example - Wildcard Certificate:

Type: CAA
Name: @
Flags: 0
Tag: issuewild
Value: "letsencrypt.org"
Example - Report Violations:

Type: CAA
Name: @
Flags: 0
Tag: iodef
Value: "mailto:security@learningmyway.space"
PTR RECORD

Purpose: Reverse DNS - Maps IP address back to domain name

Use Case: Email server reputation, network diagnostics

Example:

Forward DNS: learningmyway.space ‚Üí 192.0.2.123
Reverse DNS (PTR): 192.0.2.123 ‚Üí mail.learningmyway.space

Note: Usually managed by your hosting provider, not in your DNS
üí°

Why it matters for email: Mail servers check PTR records to verify the sender's identity. No PTR = your emails might go to spam!

SOA RECORD

Purpose: Start of Authority - Contains zone metadata and admin info

Note: Automatically created, rarely needs manual editing

Example:

Type: SOA
Primary NS: ns1.cloudflare.com
Admin Email: dns.cloudflare.com
Serial: 2024021501
Refresh: 7200
Retry: 3600
Expire: 1209600
Minimum TTL: 3600
  • Serial: Increment when zone changes (format: YYYYMMDDNN)
  • Refresh: How often secondary NS checks for updates
  • Retry: Wait time if refresh fails
  • Expire: How long secondary NS serves stale data

Common DNS Issues & Solutions

‚ùå Issue 1: Website Not Loading

üîç

Symptoms: Domain doesn't resolve, "server not found" error

Check:

  • A record exists and points to correct IP
  • Nameservers are correctly set at registrar
  • DNS propagation complete (use dnschecker.org)
  • Server is actually running and accessible

Command: dig +short learningmyway.space A

‚ùå Issue 2: Email Not Working

üìß

Symptoms: Can't receive email, emails bouncing

Check:

  • MX records exist with correct priorities
  • SPF record configured
  • DKIM records set up
  • PTR record (reverse DNS) configured

Command: dig learningmyway.space MX

‚ùå Issue 3: Slow DNS Resolution

‚è±Ô∏è

Symptoms: Website takes long to start loading

Solutions:

  • Reduce number of DNS lookups on page
  • Use a faster DNS provider (Cloudflare, Google)
  • Increase TTL values (longer caching)
  • Enable DNS prefetching in HTML

‚ùå Issue 4: Changes Not Taking Effect

‚è≥

Cause: DNS caching and propagation

Solutions:

  • Wait for TTL to expire (check old TTL value)
  • Clear local DNS cache: ipconfig /flushdns (Windows) or sudo dscacheutil -flushcache (Mac)
  • Test with: dig @8.8.8.8 learningmyway.space
  • Lower TTL before making changes (plan ahead!)

Advanced DNS Concepts


                
1. DNSSEC (DNS Security Extensions)

                
Problem: Standard DNS has no authentication - responses can be spoofed

                
Solution: DNSSEC adds cryptographic signatures to DNS records


                

                    
How DNSSEC Works:

                    
    
                        
  • Each DNS zone has a public/private key pair
    • 
                        
  • DNS records are signed with private key
    • 
                        
  • Resolvers verify signatures with public key
    • 
                        
  • Creates "chain of trust" from root to your domain
    • 
                        
  • Prevents cache poisoning and man-in-the-middle attacks
    • 
                    

                


                

                    

                        Check DNSSEC Status
                        
                    

                    
# Check if domain has DNSSEC
dig learningmyway.space +dnssec

# Verify DNSSEC validation
dig @8.8.8.8 learningmyway.space +dnssec +multi

# Check DS records (at parent zone)
dig learningmyway.space DS

                


                
2. GeoDNS (Geographic DNS)

                
Return different IP addresses based on user's location for:

                
    
                    
  • Lower latency (closer server)
    • 
                    
  • Compliance (data residency laws)
    • 
                    
  • Load distribution
    • 
                


                

                    Example GeoDNS Setup:


                    Asia users ‚Üí 192.0.2.100 (Tokyo server)

                    Europe users ‚Üí 192.0.2.200 (Frankfurt server)

3. Anycast DNS

                
Multiple servers share the same IP address. Router sends request to nearest server.

                
Used by: Cloudflare (1.1.1.1), Google (8.8.8.8), Root DNS servers
